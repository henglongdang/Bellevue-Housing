Dataset inventory

Inputs for future modeling

The project is designed to grow from manual research into a reproducible housing-policy analysis system. Each dataset has a different role: some explain law, some explain market pressure, and some explain built-environment constraints.

Housing prices

Sale and listing trends by geography, price band, and property type. Planned variables include sale price, list price, days on market, price reductions, concessions, and relist behavior.

Mortgage rates

Federal Reserve and market-rate context for buyer affordability. Rates are used as a control variable because interviews identified them as the strongest affordability force.

REET schedules

State and local rates by jurisdiction and transaction size. These create the direct transfer-cost variable.

Local tax rates

Comparable municipal cost structures across nearby communities. These allow boundary comparisons and sensitivity testing.

Interview coding results

Themes from seven professional conversations, tagged by topic, concern intensity, and agreement level.

Open house observations

Visitor concern categories, repeated questions, buyer type notes, and neighborhood comparison comments.

Policy documents

Source metadata, statutory references, capital-plan links, zoning assumptions, parking references, and policy takeaways.

Built-environment indicators

Street width, traffic exposure, walkability proxy, parking burden, and ZIP-level development-friction scores.

Correlation structure

How the variables relate

The model uses correlation-style analysis to identify patterns that deserve deeper testing. Correlation is not causation, but it helps decide which variables belong in a larger regression model.

Modeled correlation matrix

Rates show the strongest association with days-on-market pressure. REET is moderate, while zoning and parking appear as structural cost channels.

Confidence discussion

The 0.39 REET-to-days-on-market value is a modeled association used to shape the next research step. It is not evidence that REET independently causes longer listings. A production version needs transaction-level data, neighborhood fixed effects, property controls, a matched control ZIP, and enough observations to estimate confidence intervals.

Control group expansion

What would prove whether REET is the driver?

A valid test needs Bellevue compared with an equivalent high-income ZIP in a municipality without local REET over the exact same market window. That control has to match the market, not just the tax rate.

Minimum control checks

Before using a no-local-REET ZIP as a control, the study must match property mix, price band, observation window, rate period, and local tax exposure.

Current status

The present exhibit does not yet contain a transaction-level control ZIP dataset. Until that dataset is built, the correlation matrix remains a hypothesis map.

The next data release should compare Bellevue listings against a selected high-income no-local-REET municipality for the same months, using days on market, price reductions, list-to-sale ratio, property type, sale price band, and mortgage-rate period as controls.

Comparative geography

ZIP-level development friction

Geographic comparison helps prevent the project from treating Bellevue as one uniform market. Different ZIP-code areas can face different combinations of price, infrastructure, zoning, parking, and buyer demand.

Development-friction index by ZIP

The index compares modeled constraints across Bellevue ZIP-code areas. Higher scores indicate a stronger stack of cost and production barriers.

Method

The ZIP index combines five inputs: estimated price level, local transaction-cost exposure, zoning constraint score, parking burden, and infrastructure complexity. Each input is normalized to 0-100 and averaged with weights that can be adjusted in later versions.

Interpretation should be comparative. A higher score does not mean a neighborhood is worse; it means the housing system may face more production or transaction friction there.

Regression-style interpretation

What a formal model would test

The current exhibit uses transparent assumptions. A stronger future version would estimate whether local cost variables remain meaningful after controlling for rates, property type, price band, and seasonality.

Regulatory friction and monthly cost

The scatter plot frames a regression question: does higher policy friction align with higher modeled monthly cost after controls are introduced?

Candidate model

Outcome: days on market, price reduction probability, or monthly payment burden. Predictors: local REET exposure, mortgage rate, parking-cost estimate, zoning constraint score, price band, property type, and month. A full model would report coefficients, standard errors, confidence intervals, and residual diagnostics.

The exhibit's current role is to define variables and show why the model is worth building.
